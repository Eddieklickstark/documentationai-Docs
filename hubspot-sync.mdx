## Sync-Quellen

Der Sync liest:

- Kontakte aus HubSpot Contacts
- Sequenzstatus über kundenbezogene Contact-Properties
- Calls aus HubSpot Call Engagements
- E-Mails aus HubSpot Email Engagements
- protokollierte Termine aus HubSpot Meeting Engagements
- HubSpot Owners und Users für Agent-Mapping

## Kunden-Mapping

Jeder Kunde hat einen `projectKey`, zum Beispiel:

- `gigagreen`
- `bca`
- `consecur`
- `holman`
- `plan_d`
- `eyeable`
- `atmoce`

Daraus folgen Properties wie:

```text
contact_<projectKey>_status
contact_enrolled_in_sequence_date_<projectKey>
contact_unenrolled_from_sequence_date_<projectKey>
contact_meeting_booked_date_<projectKey>
```

## Sequenzen

Die Admin-Diagnose `hubspot:inspectSequenceInventory` liest echte Sequenzen über:

```text
/automation/sequences/2026-03?userId=...
```

Sie ordnet Sequenzen per Sequenzname, Projekt-Key und User-/Domain-Signal einem Kunden zu und zählt aktuelle Signale über:

```text
hs_latest_sequence_enrolled
hs_sequences_is_enrolled
```

## Historische Einschränkung

Wenn Leads aus einer HubSpot-Sequenz entfernt wurden, ist die exakte historische Mitgliedschaft pro konkreter Sequenz nicht immer nachträglich verfügbar.

Für historische Sprints sind deshalb die kundenweiten AddSales-Properties die belastbare Quelle:

```text
contact_enrolled_in_sequence_date_<projectKey>
contact_unenrolled_from_sequence_date_<projectKey>
contact_meeting_booked_date_<projectKey>
```

## Calls und E-Mails

Calls und E-Mails werden über zwei Wege gelesen:

1. Contact-Associations bereits synchronisierter Kontakte.
2. Sprintweite Suche nach erkannten Activity-Actors.

Ein Activity-Actor wird akzeptiert, wenn einer dieser Punkte passt:

- konfigurierter `customerAgent`
- E-Mail-Domain passt zu `licenseEmailDomains`
- Name/E-Mail passt zu Kunde, `projectKey` oder `sequenceNameIncludes`
- Owner einer HubSpot-Sequenz, deren Name zum Kunden passt

Geschrieben wird eine Activity nur, wenn sie mit einem synchronisierten Kundenkontakt verbunden ist, im Sprint-Zeitraum liegt und outbound/gesendet ist. Kontakte, die nur über eine Activity gefunden werden, werden nur nachgeladen, wenn sie ein kundenbezogenes Mapping-Signal tragen. Damit wird verhindert, dass derselbe Lead bei mehreren Kunden falsch zugeordnet wird.

Wenn ein akzeptierter Activity-Actor noch kein `customerAgent` ist, erzeugt Convex für den Sprint trotzdem eine eigene `sprintSequence` auf Basis von HubSpot Owner/User/E-Mail. So werden Activities von internen Sequenz-Ownern nicht mehr als unzugeordnet gesammelt.

Der Write läuft eventlimitiert in Chunks, damit große Kunden nicht an Convex Read-Limits scheitern.
