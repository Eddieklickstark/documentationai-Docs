## Voraussetzungen

- Node.js und npm
- GitHub CLI (`gh`) für Repository-Arbeit
- Convex CLI über `npx convex`
- Zugriff auf Clerk, Convex, Vercel und HubSpot für produktive Deployments

## Lokales Setup

```bash
npm install
npm run build
npm run lint
npm run test:metrics
```

Für Convex-Smoke-Tests gegen Prod:

```bash
CONVEX_DEPLOYMENT=prod:posh-ibex-42 npm run test:convex-smoke
```

## Lokale Env

Frontend-Variablen:

```text
VITE_CONVEX_URL=
VITE_CLERK_PUBLISHABLE_KEY=
```

Backend-Secrets liegen nicht im Frontend und werden in Convex gesetzt:

```text
CLERK_JWT_ISSUER_DOMAIN=
ADMIN_EMAILS=
ADMIN_EMAIL_DOMAINS=addsales.io
HUBSPOT_PRIVATE_APP_TOKEN=
RESEND_API_KEY=
RESEND_FROM_EMAIL=
```

## Development Server

```bash
npm run dev
```

## Deployment

Convex Functions:

```bash
CONVEX_DEPLOYMENT=prod:posh-ibex-42 npx convex deploy
```

Frontend:

- Vercel nutzt das GitHub-Repository.
- Build-Befehl: `npm run build`
- Output: `dist`
